Event-Driven Programming:

What is Event driven programming?
It is programming that is mainly based on the events and interactions from the user or getting triggered by events mainly saying just inputs. this could be like a mouse move, clicking, button press and etc...
Event driven programming is really suited to the programs that are really focused on user interactions and to the user changes which could be like a photoshop program. As this lets you create many options for the user like inputs and buttons and let them decide what they want. The downside of this is if you are creating program that uses GUI meaning event driven is only best for GUI programs and nothing else works that great compared to this paradigm.

Object Orientated Programming:

Object Orientated programming is focused mainly on the object and their use rather then their actions. Objects could be things like animals or humans or cars but they can have different attributes and uses where you want to call them or etc. At here you can sort things in groups or classes and if you need something specific you can call the class or object and take the data you need. You are also able to create sub classes inside the class or use techniques like inheritance which can make life easier.
Example of programming languages that are OOP: Java, C++ , C, Visual Basic and Ruby.

Loops/Iteration:

Loops are simple too it is used mostly by checking over the array or looping function until everything adds up. You can use loops in any scripting language such as javascript or PHP and etc... Mainly saying loops just repeat the sequence of instructions until specific condition is met which could be like highest key been reached on the array table. Sometimes you can accidentally create infinity loop where the program does not stops looping and could crash the program or the device itself meaning at creating loops you have to make sure you are not making infinity loops.
With iteration it got similar preference but is mostly focused on how many times the program will loop the code.

Event-Handling:

Event handling means what actions happen due to interaction such as mouse click or move of a cursor or press of the button and the event handler will handle what actions happen on the specific interaction(event). This is really great to programmers as they can change how the user interacts to the program or what is going to happen and it is all limited to imagination and creativity of the programmer.
